# 设计系统 — ARMfn Workbench

## 产品背景
- **产品定义：** AI 多 Agent 协作工作台，通过自然语言对话完成销售分析、客户管理、代码开发等任务
- **目标用户：** 产品经理、业务人员、开发者，日常与多个 AI Agent 协作
- **所属领域：** AI Agent 平台，同类产品参考 Slack、Discord、ChatGPT
- **项目类型：** Web App（React + Vite + Tailwind），聊天为核心的工作台界面
- **设计方向：** 冷峻专业（Cool Professional）— 专业、可信赖、高效，长时间使用不疲劳

## 美学方向
- **风格：** 工业/实用主义，带精致细节
- **装饰度：** 克制。不使用装饰性元素，阴影仅用于层级表达，边框清晰但不抢眼
- **氛围：** 像 Linear 或 Notion — 设计退到后面，但每个像素都是考虑过的
- **深色模式：** 必须支持，使用 CSS 变量切换，表面用层级区分而非单纯反转

## 字体

### 字体栈
| 角色 | 字体 | 说明 |
|------|------|------|
| 品牌/标题 | `DM Sans` (600/700) | 品牌名 ARMfn、页面标题、区块标题。几何感强，辨识度高 |
| 正文/UI | `ui-sans-serif, system-ui, sans-serif` | 聊天内容、表单、按钮文字。系统字体栈确保中英文混排最佳渲染 |
| 代码 | `JetBrains Mono` (400/500) | 聊天中的代码块、技术内容。支持连字，小字号清晰 |

### 加载方式
```html
<link href="https://fonts.googleapis.com/css2?family=DM+Sans:wght@500;600;700&family=JetBrains+Mono:wght@400;500&display=swap" rel="stylesheet">
```

### 字体规则
- 标题使用 `letter-spacing: -0.3px` 到 `-0.5px`，正文不调字间距
- 代码块使用 `font-variant-numeric: tabular-nums`
- 数据表格列使用 `tabular-nums` 确保数字对齐
- 行宽上限 640px（正文），聊天气泡最大 72% 容器宽度

## 色彩

### 设计原则
- **策略：** 均衡。Sky Blue 主色承载品牌和交互，语义色覆盖所有状态反馈
- **中性色：** Slate 系列（冷蓝灰调），和主色协调
- **深色模式：** 主色提亮一档，语义色提亮一档，表面用 Slate 暗色阶梯

### 色彩变量

#### 主色
| 变量名 | 浅色模式 | 深色模式 | 用途 |
|--------|---------|---------|------|
| `--primary` | #0EA5E9 | #38BDF8 | 按钮、链接、选中态、品牌色 |
| `--primary-hover` | #0284C7 | #0EA5E9 | 按钮悬停、链接悬停 |
| `--primary-light` | rgba(14,165,233,0.12) | rgba(56,189,248,0.15) | 选中背景、徽标底色、hover 底色 |

#### 语义色
| 变量名 | 浅色模式 | 深色模式 | 用途 |
|--------|---------|---------|------|
| `--success` | #10B981 | #34D399 | Agent 在线、操作成功、确认 |
| `--error` | #F43F5E | #FB7185 | 离线、错误提示、危险操作 |
| `--warning` | #F59E0B | #FBBF24 | 处理中、额度不足、注意事项 |
| `--info` | #38BDF8 | #7DD3FC | 已连接、提示信息 |
| `--approval` | #8B5CF6 | #A78BFA | 审批卡片色条、审批状态标签（P2 预留） |

#### 中性色（表面和文字）
| 变量名 | 浅色模式 | 深色模式 | 用途 |
|--------|---------|---------|------|
| `--bg` | #F8FAFC | #0F172A | 页面背景 |
| `--surface` | #FFFFFF | #1E293B | 卡片、面板、聊天气泡 |
| `--surface-2` | #F1F5F9 | #334155 | 悬停底色、代码块、次级表面 |
| `--border` | #E2E8F0 | #334155 | 所有边框 |
| `--text-1` | #0F172A | #F1F5F9 | 标题、正文 |

| `--text-3` | #94A3B8 | #64748B | 占位符、禁用态 |

### 色彩使用规则
- 语义色用于徽标/状态点/提醒时，背景取 8-12% 透明度，文字取深一档实色
- 深色模式下语义色统一提亮一档（500 → 300/400），避免暗背景上看不清
- 不在大面积区域使用高饱和色，主色仅用于按钮和小面积强调
- Ghost 按钮在深色模式下必须有 `rgba(255,255,255,0.06)` 底色，确保可见

## 布局

### 整体结构
- **模式：** 网格规整（Grid-disciplined）
- **核心布局：** 三栏——会话侧边栏（240px）/ 聊天主区域（弹性）/ 右侧面板（260px）
- **内容最大宽度：** 聊天区域不设上限，正文内容区 640px
- **断点：**
  - 移动端 (≤640px)：单栏，底部导航栏
  - 平板 (641-1024px)：双栏，隐藏右侧面板
  - 桌面 (≥1025px)：三栏完整布局

### 会话列表规则
- 每个会话项之间用 1px 分隔线（`border-bottom: 1px solid var(--border)`）
- 选中态使用 `var(--primary-light)` 背景
- 头像必须保留（图片或首字缩写），纯文字列表辨识度差
- 不显示 Agent 内部 slug（如 `yangwenxing-sales-bp`），只展示显示名称

### 聊天消息规则
- 用户消息：靠右，背景 `var(--surface-2)`，无边框
- AI 消息：靠左，背景 `var(--surface)`，1px `var(--border)` 边框
- **头像和发送者名称在气泡外部**（上方），气泡仅包裹消息正文和附属内容（代码块、工具调用状态）
- 气泡最大宽度 72%
- 新消息入场动画：`translateY(8px)` + `opacity: 0` → 正常位置，200ms ease-out

### Agent 思维链展示规则

Agent 消息包含三层信息，视觉层级：**最终回答 > 思考过程 > 工具执行**

#### 三层信息规格

| 层级 | 字号 | 颜色 | 默认状态 | 用途 |
|------|------|------|---------|------|
| 最终回答 | base (14px) | `var(--text-1)` | 始终展示 | Agent 给用户的最终结果，markdown 渲染 |
| 思考过程 | sm (13px) | `var(--text-2)` | 流式展开，完成后**自动折叠** | Agent 的自然语言推理过程 |
| 工具执行 | xs (12px) | `var(--text-3)` | 始终折叠，一行自然语言描述 | Agent 调用的工具，如"读取了文件 config.ts" |

#### 交互状态

| 状态 | 思考过程 | 工具调用 | 最终回答 |
|------|---------|---------|---------|
| STREAMING | 展开，实时更新，尾部光标闪烁 | 一行描述 + dot 脉冲动画 | 尚未出现 |
| COMPLETE | 自动折叠，点击展开 | 静态一行描述，点击可展开详情 | 展开，markdown，最突出 |
| ERROR | 保持展开 | 错误工具行标红 | 错误提示 |
| EMPTY（刚开始） | 不显示 | 不显示 | "思考中..." 三点脉冲动画 |

#### 设计约束

- 思考过程：无装饰（不使用左边框竖线，违反 AI Slop 黑名单 #8），纯文本段落
- 工具执行：使用自然语言描述（"正在读取文件..."），不暴露工具名、JSON 参数、执行结果
- 工具详情保留隐藏入口：点击工具行可展开原始数据，供开发者 debug
- 折叠按钮文案："思考过程" / "收起思考过程"
- 折叠按钮触控目标 ≥ 44px，添加 `aria-expanded` 属性
- 最终回答前与工具/思考区域间距 8px

## 圆角

### 圆角层级
| Token | 值 | 用途 |
|-------|-----|------|
| sm | 4px | 徽标、代码行内、小按钮 |
| md | 8px | 按钮、输入框、卡片、气泡 |
| lg | 12px | 弹窗、面板、大容器 |
| full | 9999px | 头像、药丸标签、状态点 |

### 圆角规则
- 不要全局使用相同的大圆角（这是 AI slop 特征）
- 嵌套元素：内圆角 = 外圆角 - 间距
- 登录卡片等独立容器用 lg (12px)，内部元素用 md (8px) 或 sm (4px)

## 动效

### 设计原则
- **策略：** 功能导向。每个动画必须回答"它帮用户理解了什么？"
- 不使用装饰性动画、弹跳效果、超过 500ms 的过渡
- 尊重 `prefers-reduced-motion: reduce`

### 缓动函数
| 场景 | 函数 |
|------|------|
| 进入 | `ease-out` 或 `cubic-bezier(0.16, 1, 0.3, 1)` |
| 退出 | `ease-in` |
| 移动 | `ease-in-out` |

### 时长阶梯
| 级别 | 时长 | 用途 |
|------|------|------|
| 微交互 | 100-150ms | hover 变色、toggle 切换、focus 状态 |
| 内容过渡 | 200-300ms | 新消息出现、面板展开/收起、页面切换 |
| 状态变化 | 300-500ms | 加载动画、进度指示 |

### 优先实现的动效
1. 新消息滑入（`translateY(8px)` + fade，200ms ease-out）
2. 侧面板展开/收起（宽度 + opacity，250ms ease-out）
3. 所有交互元素的 hover/focus 色彩过渡（150ms ease-out）
4. 加载骨架屏闪烁动画

### 绝对不做
- 页面整体入场动画
- 弹跳/回弹缓动
- 任何超过 700ms 的动效
- `transition: all`（必须列出具体属性）
- 对 width/height/top/left 做动画（只用 transform 和 opacity）

## 触控目标
- 所有可交互元素最小 44x44px（包括 padding 在内的点击区域）
- 侧边栏图标按钮不得小于 44px（当前是 24px，必须修复）
- 移动端输入框使用正确的 `inputmode` 属性

## 无障碍
- 正文文字对比度 ≥ 4.5:1 (WCAG AA)
- 大字 (≥18px) 对比度 ≥ 3:1
- UI 组件对比度 ≥ 3:1
- 所有交互元素必须有 `focus-visible` 样式，不得使用 `outline: none` 不提供替代
- 不使用纯颜色编码信息（状态点旁必须有文字标签）

## 组件规范 — 任务卡片

### 卡片容器
- 背景：`var(--surface)`
- 边框：`1px solid var(--border)`
- 圆角：`md (8px)`
- 内边距：`md (12px)` 顶部/左右，`base (16px)` 底部
- 最大宽度：对话内 = 气泡 72%，工作台 = 100%
- 左侧语义色条：3px 宽，按 cardType 映射

### 语义色条映射
| cardType | 色条颜色 | 图标 | 标签文字 |
|----------|----------|------|----------|
| visit_report | `var(--success)` | ✎ | 拜访汇报 |
| action_suggestion | `var(--primary)` | ▶ | 行动建议 |
| insight | `var(--info)` | ◆ | 分析洞察 |
| escalation | `var(--warning)` | ▲ | 风险升级 |
| approval | `var(--approval)` | ● | 待审批 |
| sync_result | `var(--success)` | ✓ | 同步结果 |

注意：语义色条不是装饰（AI Slop #8），而是有明确映射的类型标识，配合文字标签满足无障碍要求。

### 状态徽标
- 字号 12px，font-weight 600
- 背景取语义色 8-12% 透明度，文字取对应语义色深一档
- 圆角 `sm (4px)`

### 类型标签
- 字号 12px，圆角 `sm (4px)`
- 图标 + 文字双重编码（满足无障碍：不使用纯颜色编码信息）

### 卡片信息层级（从上到下）
1. 状态徽标 + 类型标签 + 优先级
2. 标题（16px DM Sans 600，`letter-spacing: -0.3px`）
3. 摘要/结论（14px 系统字体，max 3 行 clamp）
4. 关键依据（13px `var(--text-2)`，左侧 2px `var(--border)` 线）
5. 元数据行（12px `var(--text-3)`：置信度 · 覆盖率 · 时间）
6. 建议动作（14px，`var(--surface-2)` 背景，可编辑）
7. 操作按钮组（主按钮 `var(--primary)` + ghost 次按钮 + danger 按钮）

### 已完成卡片
- 整体 `opacity: 0.6`
- 标题颜色降为 `var(--text-3)`

## 组件规范 — 底部 Sheet（移动端）

仅 ≤640px 视口激活，替代右侧面板。
- 顶部圆角：`lg (12px)`，底部无圆角
- 背景：`var(--surface)`
- 阴影：`0 -4px 16px rgba(0,0,0,0.12)`
- 最大高度：视口 75%
- 拖拽指示条：40x4px，`var(--text-3)` 色，圆角 `full`
- 打开动画：从底部滑入，300ms ease-out
- 关闭动画：向下滑出，200ms ease-in
- 遮罩层：`rgba(0,0,0,0.4)`，点击关闭

## 组件规范 — 反馈表单（深色模式注意）

- 表单容器：`var(--surface-2)` + `1px solid var(--border)`（深色下需要边框辅助层级区分）
- 标签文字：`var(--text-1)` + font-weight 500（不能用 text-2/text-3，深色下对比度不够）
- 输入框背景：`var(--bg)`（三级层次：card surface > form surface-2 > input bg）
- 选项按钮未选中：`var(--bg)` 背景 + `var(--text-1)` 文字
- 选项按钮选中：语义色 15% 透明度背景 + 语义色文字

## AI Slop 黑名单（不允许出现）
1. 紫色/靛蓝渐变背景
2. 三列 icon + 标题 + 描述的 feature grid
3. 圆形彩色背景里放 icon
4. 全部居中对齐（`text-align: center` 不应是默认值）
5. 所有元素统一大圆角
6. 装饰性 blob、波浪线、浮动圆形
7. emoji 作为设计元素
8. 卡片左边框彩色条纹
9. "欢迎使用..."、"一站式..."等通用文案
10. 千篇一律的 hero → features → testimonials → pricing 节奏
