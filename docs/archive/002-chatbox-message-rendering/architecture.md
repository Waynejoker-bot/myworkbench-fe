## 二、消息数据结构转换

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                        Message Station 原始消息                              │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  用户发送的消息:                                                             │
│  RawMessage {                                                                │
│    id: 1,                                                                    │
│    message_id: "msg-user-001",     // 消息唯一标识                           │
│    round_id: "msg-user-001",       // 用户消息的 round_id = message_id      │
│    from: "user-zc",                                                          │
│    to: "agent-a",                                                            │
│    seq: 0,                     status: 3 (END),                              │
│    payload: "帮我分析一下数据",                                               │
│  }                                                                            │
│                                                                              │
│  Agent 返回的消息（流式）:                                                    │
│  RawMessage {                                                                │
│    id: 2,                                                                    │
│    message_id: "msg-agent-001",     // Agent 消息的唯一标识                   │
│    round_id: "msg-user-001",       // round_id = 用户消息的 message_id       │
│    from: "agent-a",                // 表示回复这条用户消息                    │
│    to: "user-zc",                                                             │
│    seq: 0,                     status: 1 (START),                            │
│    payload: "好的，我来",                                                      │
│  }                                                                            │
│  RawMessage {                                                                │
│    id: 3,                                                                    │
│    message_id: "msg-agent-001",     // 同一条消息                             │
│    round_id: "msg-user-001",       // 同一个 round_id                        │
│    seq: 1,                     status: 2 (CHUNK),                            │
│    payload: "分析这个数据...",                                                  │
│  }                                                                            │
│  RawMessage {                                                                │
│    id: 4,                                                                    │
│    message_id: "msg-agent-001",                                              │

│  }                                                                            │
│                                                                              │

                                    │

│                           聚合后的 UI 消息                                    │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │

│  }                                                                            │
│                                                                              │
│  Agent 消息（聚合后）:                                                        │
│  UIMessage {                                                                 │
│    id: "msg-agent-001",                                                      │
│    replyToId: "msg-user-001",       // 回复用户消息 (引用样式)               │
│    role: "assistant",                                                        │

│      { seq: 0, status: START, content: "好的，我来" },                       │

│      { seq: 2, status: END, content: "完成！" }                              │

│    messageStatus: END,                                                        │
│  }                                                                            │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
                                    │

┌──────────────────────────────────────────────────────────────────────────────┐

│    ],                                                                        │
│  }                                                                            │
│                                                                              │
```

## 三、关键概念说明

### 3.1 message_id vs round_id

| 字段 | 用途 | 示例 |
|------|------|------|
| `message_id` | 消息唯一标识，用于聚合同一条流式消息 | "msg-agent-001" |
| `round_id` | 回复关系标识，Agent 响应的 round_id = 用户消息的 message_id | "msg-user-001" |

### 3.2 回复关系

```
用户消息: message_id = "msg-001", round_id = "msg-001"
                    │
                    │ round_id
                    ▼
Agent 响应: message_id = "msg-002", round_id = "msg-001"
                                          ↑
                                   replyToId 指向 "msg-001"
```

## 五、消息状态机

```
                    ┌─────────────────┐
                    │   STATUS_START  │
                    │   (seq=0)       │
                    │  "Hello"        │
                    └────────┬────────┘
                             │
                             │ 接收下一个 CHUNK
                             ▼
                    ┌─────────────────┐
                    │  STATUS_CHUNK   │◄─────┐
                    │   (seq=1,2,3...) │      │
                    │  " world"       │      │
                    └────────┬────────┘      │
                             │               │
                             │ 接收 END 或    │
                             │ 更多 CHUNK     │
                             ▼               │
                    ┌─────────────────┐      │
                    │   STATUS_END    │      │
                    │   (seq=N)       │      │
                    │  "!"            │      │
                    └─────────────────┘      │
                             │               │
                             │               │
                    完成/最终状态              │
                             │               │
                             └───────────────┘

                    ┌─────────────────┐
                    │  STATUS_ERROR   │
                    │   (任何时候)    │
                    │  错误信息        │
                    └─────────────────┘
```
