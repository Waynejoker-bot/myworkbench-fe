# Workbench 重大重构 - 需求文档

## 1. 项目背景

### 1.1 当前系统状态
MyWorkbench-FE 是一个基于 Shadow DOM 的动态组件工作台系统，已实现：
- 组件动态加载（通过 manifest.json）
- Shadow DOM 样式隔离
- ChatBox 与组件之间的双向通信
- 组件注册表和组件市场
- 基础的 WorkbenchContainer 容器

### 1.2 存在的问题
在实际使用过程中发现以下痛点：

1. **组件发现困难**
   - 用户无法直观看到所有可用组件
   - 需要通过下拉选择器才能看到组件列表
   - 新用户不知道系统中有哪些组件

2. **显示区域受限**
   - 组件被限制在固定宽度（560px）的小框中
   - 无法充分利用屏幕空间
   - 大型组件（如文件浏览器）体验不佳

3. **多任务处理不便**
   - 无法同时打开多个组件
   - 无法在多个组件间快速切换
   - 频繁打开/关闭组件影响工作效率

### 1.3 用户需求概述
需要对右侧 Workbench 进行重大调整：

1. **组件列表页** - 作为 Workbench 的初始视图，展示所有可用组件
2. **全屏组件加载** - 点击组件后全屏加载，充分利用空间
3. **多 Tab 支持** - 支持在多个 Tab 间切换，便于多任务处理

## 2. 功能需求

### 2.1 组件列表视图 (FR-001)

**优先级**: P0 (必须有)

**描述**:
Workbench 打开时默认显示组件列表视图，而不是空白状态。

**功能细节**:
1. **组件展示**
   - 以卡片形式展示所有可用组件
   - 每个卡片显示：组件图标、名称、描述、版本号
   - 支持网格布局（自适应屏幕宽度）

2. **搜索功能**
   - 提供搜索框，支持实时搜索
   - 搜索范围：组件名称、描述
   - 搜索结果高亮显示

3. **分类筛选**
   - 支持按分类筛选组件
   - 预定义分类：productivity, files, data, communication, utilities, developer, media
   - 分类通过 manifest.json 配置

4. **最近使用**
   - 显示最近使用的组件（最多 10 个）
   - 按使用时间倒序排列
   - 点击可快速打开

5. **收藏功能**
   - 支持收藏常用组件
   - 收藏的组件显示星标
   - 收藏状态持久化保存

**验收标准**:
- [ ] 打开 Workbench 时默认显示组件列表
- [ ] 显示所有已注册的组件
- [ ] 搜索框输入时实时过滤组件
- [ ] 分类下拉菜单正确筛选组件
- [ ] 最近使用的组件正确显示
- [ ] 收藏按钮可以切换收藏状态
- [ ] 刷新页面后收藏状态保持

---

### 2.2 Tab 系统 (FR-002)

**优先级**: P0 (必须有)

**描述**:
支持打开多个组件 Tab，并在 Tab 间切换。

**功能细节**:
1. **Tab 打开**
   - 点击组件卡片打开新 Tab
   - Tab 标题显示组件名称
   - Tab 显示加载状态
   - 组件加载完成后显示内容

2. **Tab 切换**
   - 点击 Tab 标题切换到对应组件
   - 当前激活的 Tab 高亮显示
   - 未激活的 Tab 保持状态（不卸载）
   - 切换时触发组件的 `onVisibilityChange` 回调

3. **Tab 关闭**
   - Tab 上显示关闭按钮（×）
   - 点击关闭按钮卸载组件
   - 最后一个 Tab 关闭后返回组件列表视图
   - 支持通过键盘快捷键关闭（Ctrl+W）

4. **Tab 限制策略**
   - 通过 manifest.json 配置 Tab 策略
   - **单例模式** (singleton): 同一组件只能打开一个实例
     - 如果已存在该组件的 Tab，切换到该 Tab
     - 示例：todolist
   - **多例模式** (multiple): 可以打开多个实例
     - 可选配置 `maxInstances` 限制最大实例数
     - 示例：file-browser
   - 未配置时默认为多例模式

5. **Tab 新建**
   - Tab 栏右侧显示"+"按钮
   - 点击返回组件列表视图

**验收标准**:
- [ ] 点击组件卡片打开新 Tab
- [ ] Tab 栏显示所有打开的 Tab
- [ ] 点击 Tab 标题可以切换
- [ ] 点击关闭按钮可以关闭 Tab
- [ ] 最后一个 Tab 关闭后返回组件列表
- [ ] 单例组件只能打开一个实例
- [ ] 多例组件可以打开多个实例
- [ ] 达到 `maxInstances` 限制时无法继续打开
- [ ] Tab 状态在切换时正确保持

### 2.3 全屏组件加载 (FR-003)

**优先级**: P0 (必须有)

**描述**:
组件在 Tab 中全屏加载，充分利用 Workbench 区域。

**功能细节**:
1. **全屏显示**
   - 组件占据整个 Tab 内容区域
   - 移除原有的固定宽度限制
   - 组件可以自适应容器尺寸

2. **响应式设计**
   - 组件响应容器尺寸变化
   - 调用组件的 `onResize` 回调
   - 支持通过拖拽调整 Workbench 宽度

3. **组件通信**
   - 保持现有的双向通信机制
   - 支持 Host API 调用
   - 支持消息订阅

**验收标准**:
- [ ] 组件在全屏模式下正确显示
- [ ] 组件可以自适应容器尺寸
- [ ] 调整 Workbench 宽度时组件正确响应
- [ ] 组件可以正常调用 Host API
- [ ] 组件可以正常订阅消息

---

### 2.4 持久化存储 (FR-004)

**优先级**: P1 (应该有)

**描述**:
保存用户偏好和使用历史。

**功能细节**:
1. **最近使用记录**
   - 记录最近使用的 10 个组件
   - 按使用时间排序
   - 显示在组件列表顶部

2. **收藏组件**
   - 用户可以收藏常用组件
   - 收藏状态持久化
   - 收藏的组件优先显示

3. **Tab 状态恢复** (可选)
   - 保存当前打开的 Tab 配置
   - 刷新页面后恢复 Tab 状态
   - 提供"恢复上次工作区"选项

**存储方案**:
- 使用 localStorage
- 数据结构：
```json
{
  "recentComponents": ["file-browser", "todolist"],
  "favoriteComponents": ["file-browser", "calculator"],
  "savedTabs": [...] // 可选
}
```

**验收标准**:
- [ ] 使用的组件自动添加到最近使用记录
- [ ] 最近使用记录最多保存 10 个
- [ ] 收藏的组件显示星标
- [ ] 刷新页面后最近使用记录保持
- [ ] 刷新页面后收藏状态保持

---

### 2.5 组件清单扩展 (FR-005)

**优先级**: P1 (应该有)

**描述**:
扩展 manifest.json 配置以支持新功能。

**新增字段**:
```json
{
  "name": "my-component",
  "version": "1.0.0",

  // Tab 策略
  "tabStrategy": {
    "mode": "singleton",        // singleton | multiple
    "maxInstances": 1,          // 多例模式下的最大实例数
    "defaultTitle": "{{name}}", // Tab 标题模板
    "allowClose": true          // 是否允许关闭
  },

  // 分类和标签
  "category": "productivity",   // 组件分类
  "tags": ["todo", "task"],     // 标签

  // 默认视图模式
  "viewMode": "fullscreen"      // fullscreen | panel
}
```

**向后兼容**:
- 所有新字段均为可选
- 未配置时使用默认值

**验收标准**:
- [ ] manifest.json 可以配置 tabStrategy
- [ ] 未配置 tabStrategy 的组件使用默认值
- [ ] singleton 模式正确限制实例数
- [ ] multiple 模式正确打开多个实例
- [ ] maxInstances 限制正确生效

## 3. 非功能需求

### 3.1 性能需求 (NFR-001)

**优先级**: P1 (应该有)

**指标**:
1. **组件列表加载**
   - 首次加载时间 < 500ms
   - 后续打开时间 < 100ms（使用缓存）

2. **Tab 操作**
   - 打开 Tab 时间 < 2s（包含组件加载）
   - 切换 Tab 响应时间 < 100ms
   - 关闭 Tab 时间 < 200ms

3. **内存占用**
   - 同时打开 10 个 Tab 时内存占用 < 500MB
   - 单个组件内存占用 < 50MB

4. **优化措施**
   - 组件列表使用虚拟滚动（当组件 > 50 时）
   - 未激活的 Tab 延迟加载或卸载
   - 组件资源预加载和缓存

---

### 3.2 可扩展性需求 (NFR-002)

**优先级**: P1 (应该有)

**描述**:
系统架构应支持未来功能扩展。

**预留扩展点**:
1. **Tab 拖拽排序**

   - 支持 Tab 拖拽到新窗口

2. **Tab 分组**
   - 支持 Tab 分组显示
   - 支持分组折叠/展开

3. **工作区保存/恢复**
   - 保存完整工作区配置
   - 支持多个工作区配置

4. **组件间通信**
   - 允许不同 Tab 的组件直接通信
   - 提供组件间消息总线

---

### 3.3 兼容性需求 (NFR-003)

**优先级**: P0 (必须有)

**描述**:

**优先级**: P1 (应该有)

**描述**:

## 6. 风险评估

### 6.1 性能风险
**风险描述**: 同时打开多个 Tab 可能导致内存占用过高

**影响**: 中等
**概率**: 中等

**缓解措施**:
- 限制最大 Tab 数量（如 20 个）
- 未激活的 Tab 自动卸载
- 提供内存使用监控
- 优化组件加载和卸载逻辑

---

### 6.2 兼容性风险
**风险描述**: 现有组件可能不适应新的 Tab 交互模式

**影响**: 高
**概率**: 低

**缓解措施**:
- 保持向后兼容
- 提供渐进式迁移路径
- 充分测试现有组件
- 提供组件适配指南

---

### 6.3 用户体验风险
**风险描述**: 功能复杂度增加可能影响易用性

**影响**: 中等
**概率**: 低

**缓解措施**:
- 提供简洁的默认视图（组件列表）
- 添加引导提示
- 收集用户反馈持续优化
- 进行可用性测试
